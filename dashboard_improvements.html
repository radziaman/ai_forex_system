1️⃣ Walk-Forward Validation

What is it?

Tests if models GENERALIZE to unseen data (not just memorizing training data).

Method

Window Training Period Test Period Status
Window 1 2019-2024 (6 years) 2025 (unseen data) ⚠ Debugging (0 trades)
Window 2 2020-2025 (6 years) 2026 Q1 (unseen) ⚠ Debugging (0 trades)

Why it Matters

  • Without validation: Models might be overfitting (memorizing) instead of learning patterns
  • With validation: Prove models work on data they've NEVER seen
  • Target: >0% return on unseen data = models generalize well

3️⃣ Dynamic Position Sizing (Kelly Criterion)

What is it?

Replaces fixed $10k exposure with mathematically optimal position sizes.

Kelly Criterion Formula

# Kelly Fraction = (Win_Rate × Win/Loss_Ratio - Loss_Rate) / Win/Loss_Ratio # Example: # Win Rate = 60% # Avg Win = 3% # Avg Loss = 2% # Win/Loss Ratio = 3/2 = 1.5 # Kelly = (0.6 × 1.5 - 0.4) / 1.5 = 0.4 = 40% # Apply safeguards: kelly = min(kelly, 0.25) # Cap at 25% (half-Kelly safer) kelly = kelly × confidence # Scale by model confidence kelly = kelly × volatility_factor # Reduce if high volatility

Comparison: Fixed vs Dynamic

Metric Fixed ($10k always) Dynamic (Kelly-based) Improvement
Position Size Always $10,000 $5,000 - $25,000 (scaled) 2.5x better risk mgmt
High Confidence $10,000 $20,000 (2x larger) Capture more profit
Low Confidence $10,000 $5,000 (0.5x smaller) Reduce risk
High Volatility $10,000 $6,000 (0.6x smaller) Adapt to risk
Win Rate 60%+ $10,000 $15,000 (1.5x larger) Exploit edge
